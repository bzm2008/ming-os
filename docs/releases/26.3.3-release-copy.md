# Ming OS 26.3.3 官网与 GitHub Release 文案

> 历史预览文案：官网和 GitHub 发布入口应使用 26.4.0。

本文件是发布侧可直接审阅的文案，不包含未经发布流水线确认的产物地址、SHA256 或公钥指纹。

## 官网说明

### 标题

Ming OS 26.3.3：更安全的单入口事务型 OTA

### 正文

Ming OS 26.3.3 将系统更新统一放入 Ming 设置中心。用户只需点击“检查更新”，发现完成签名校验的版本后点击“立即更新”。系统根据事务状态展示暂存、重启、健康检查、完成或自动回滚，不会把“已下载”误报为“已更新”。

26.3.2 用户首次使用单分区事务型 OTA 前，需要安装官方签名的 bootstrap。下载页会同时提供 bootstrap、SHA256、 detached signature 和公钥指纹；安装前必须完成两项校验。没有 bootstrap、空间不足、签名失败或健康检查失败时，系统会明确拒绝或自动回滚，不要求用户绕过安全门禁。

设置中心还可以导出脱敏的事务诊断，便于支持人员定位问题。recovery ISO 仍要求独立备份介质，事务型 OTA 不会改变该保护规则。

### 页面按钮

- 设置中心：`检查更新`
- 发现可执行更新：`立即更新`
- 事务进行中：`等待完成`（不可重复提交）
- 诊断页：`导出更新诊断`

## GitHub Release

### Release title

`Ming OS 26.3.3`

### Release body

Ming OS 26.3.3 收敛了系统更新入口，并为事务型 OTA 增加了清晰的状态和失败原因展示。

Highlights:

- Ming 设置中心保留唯一更新入口：检查更新 → 立即更新。
- 结构化展示 `new`、`verified`、`staged`、`armed`、`pending_health`、`committed` 和回滚状态。
- 明确提示 bootstrap 缺失、空间不足、manifest/artifact 签名失败和健康检查失败。
- 提供脱敏事务诊断导出，不读取或上传用户文件内容。
- 26.3.2 继续使用一次性官方签名 bootstrap；recovery ISO 的独立备份介质门禁保持不变。

Before upgrading from 26.3.2, read the official bootstrap instructions and verify the package checksum and detached signature. Do not disable signature, space, backup, or rollback checks.

发布流水线必须在发布前填入并复核以下资产，不能用示例值替代：

- ISO/VHDX 及其 SHA256（如本次发布包含安装介质）。
