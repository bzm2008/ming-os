# Ming OS 26.4.0 正式版发布说明

> 发布状态：正式候选验证中；官网和事务型 OTA 继续保持旧预览版/`delivery:none`，待签名发布材料和升级回归全部通过后切换。

## 更新摘要

Ming OS 26.4.0 跳过 26.3.3，继续使用 Debian 13、Linux 6.12、NetworkManager、PulseAudio 和 X11。设置中心保留唯一的“检查更新/立即更新”入口；26.3.2 的事务型 OTA 只有在签名 manifest、content index、payload 和 bootstrap 齐全后才会开放。

本版本修复了设置密码后屏保仍使用默认认证、联网后校时服务未重试、Thunar 右键菜单缺少 DEB 安装入口等问题，并补充了安装、无线、软件安装和 OTA 的回归门禁。

## 26.3.2 / 26.3.3 / 26.4.0 预览版 -> 26.4.0 正式版

1. 在 26.3.2 设置中心检查更新；没有 bootstrap 时只显示官方签名 bootstrap 的下载与校验说明。
2. 安装并验证 bootstrap 的 SHA256、detached signature 和公钥指纹。
3. 再次点击“检查更新”，确认签名 manifest 的 `from_versions` 为 `["26.3.2", "26.3.3", "26.4.0"]`、事务目标版本为 `26.4.0.1`、架构为 `amd64`。
4. 签名材料发布并通过发现门禁后，点击唯一的“立即更新”按钮。系统自动完成下载、内容索引校验、空间检查、用户数据备份、暂存、一次性启动和健康确认；失败或断电时自动回滚。当前线上 discovery 返回 `delivery:none`，不会误触发不完整升级。

单分区事务型 OTA 不需要外接硬盘，也不调用 Calamares、分区工具、格式化或调整分区。Recovery ISO 的独立备份介质限制保持不变。

## 重点回归

- 新密码通过 `ming-account-control` 读回，并从 `nopasswdlogin` 组移除后才允许报告成功；屏保使用新密码认证。
- `systemd-timesyncd` 是必需运行时依赖，NetworkManager 连接事件和延迟定时器均会触发有界同步，不阻塞图形登录。
- Thunar 最终 `uca.xml` 同时匹配 `.deb/.DEB`，MIME fallback 使用相同的验证安装器并刷新应用入口。
- 无线状态区分无硬件、驱动/固件缺失、rfkill 阻断和可扫描；VirtualBox 无无线硬件不误报驱动失败。
- OTA 只消费结构化 JSON 状态；签名、版本、架构、空间、/home 保留和回滚门禁不因 UI 改动而弱化。

## 事务 OTA 发布前置条件

正式 ISO 尚未构建，最终大小和 SHA256 必须由完整模块重放后的产物填写，不复用预览版哈希。
事务 OTA 仍需由发布负责人生成并审核 26.3.2 bootstrap、签名 manifest、payload、content index 和 detached signatures，完成预览版及真机回归后再启用 discovery。
