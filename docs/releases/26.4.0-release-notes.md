# Ming OS 26.4.0 发布说明

> 发布状态：代码与升级元数据已准备，ISO 构建与发布需在最终审查后进行。

## 更新摘要

Ming OS 26.4.0 跳过 26.3.3，继续使用 Debian 13、Linux 6.12、NetworkManager、PulseAudio 和 X11。设置中心保留唯一的“检查更新/立即更新”入口，26.3.2 用户安装一次官方签名 bootstrap 后即可走事务型 OTA。

本版本修复了设置密码后屏保仍使用默认认证、联网后校时服务未重试、Thunar 右键菜单缺少 DEB 安装入口等问题，并补充了安装、无线、软件安装和 OTA 的回归门禁。

## 26.3.2 -> 26.4.0

1. 在 26.3.2 设置中心检查更新；没有 bootstrap 时只显示官方签名 bootstrap 的下载与校验说明。
2. 安装并验证 bootstrap 的 SHA256、detached signature 和公钥指纹。
3. 再次点击“检查更新”，确认签名 manifest 的 `from_versions` 包含 `26.3.2`、目标版本为 `26.4.0`、架构为 `amd64`。
4. 点击唯一的“立即更新”按钮。系统自动完成下载、内容索引校验、空间检查、用户数据备份、暂存、一次性启动和健康确认；失败或断电时自动回滚。

单分区事务型 OTA 不需要外接硬盘，也不调用 Calamares、分区工具、格式化或调整分区。Recovery ISO 的独立备份介质限制保持不变。

## 重点回归

- 新密码通过 `ming-account-control` 读回，并从 `nopasswdlogin` 组移除后才允许报告成功；屏保使用新密码认证。
- `systemd-timesyncd` 是必需运行时依赖，NetworkManager 连接事件和延迟定时器均会触发有界同步，不阻塞图形登录。
- Thunar 最终 `uca.xml` 同时匹配 `.deb/.DEB`，MIME fallback 使用相同的验证安装器并刷新应用入口。
- 无线状态区分无硬件、驱动/固件缺失、rfkill 阻断和可扫描；VirtualBox 无无线硬件不误报驱动失败。
- OTA 只消费结构化 JSON 状态；签名、版本、架构、空间、/home 保留和回滚门禁不因 UI 改动而弱化。

## 发布前置条件

本版本暂不构建 ISO。发布前必须完成 BIOS/UEFI 全盘和分区安装、Ventoy Live 桌面、真实无线连接、DEB 安装启动、26.3.2 bootstrap 以及 26.3.2 -> 26.4.0 事务 OTA 的真机回归，并生成最终签名 manifest、payload、content index、校验值和公钥指纹。
