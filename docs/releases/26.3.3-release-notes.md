# Ming OS 26.3.3 发布说明

> 历史预览文档：26.3.3 不作为公开版本，当前发布目标为 26.4.0。

> 发布状态：文档模板。只有在签名 manifest、payload、bootstrap 和最终 SHA256 写入官网与 GitHub Release 后，才能将本页标记为正式发布。

## 更新摘要

Ming OS 26.3.3 将事务型 OTA 的用户入口收敛到 Ming 设置中心，并继续保留 26.3.2 的一次性官方 bootstrap 路径。设置中心只提供一个更新入口：没有更新时显示“检查更新”，发现已完成签名校验的更新时显示“立即更新”。不会再出现独立的桌面更新程序。

本版本的界面只读取 `ming-update status --json`、`ming-update check --json` 和 `ming-update apply --release-id ID --manifest-sha256 HASH --json` 的结构化结果，不解析终端进度文本。

## 更新状态

| 事务状态 | 设置中心显示 | 用户动作 |
| --- | --- | --- |
| `new` / `verified` | 准备更新 / 已完成签名校验，准备暂存 | 仅在 CLI 明确允许时点击“立即更新” |
| `staging` / `staged` | 正在暂存 / 已下载并完成校验，等待重启确认 | 等待事务完成，不重复提交 |
| `armed` / `booting` / `pending_health` | 已安排下一次启动 / 正在启动候选系统，等待健康检查 | 不关机断电，等待健康检查 |
| `committing` / `committed` | 正在确认 / 更新已完成 | `committed` 后可再次检查更新 |
| `rollback_armed` / `rolling_back` / `rolled_back` | 已安排回滚 / 正在回滚 / 更新未通过健康检查，已自动回滚 | 保留日志，确认上一版本可用 |

“已下载并完成校验”不等于“已更新”。只有事务状态为 `committed` 时才显示“更新已完成”。

## 安全拒绝提示

设置中心会直接显示稳定错误码对应的中文原因：

- `E_BOOTSTRAP_REQUIRED`：先安装官方签名 bootstrap；不把 recovery ISO 当作同盘升级替代品。
- `E_SPACE`：可用空间不足，安全拒绝；释放空间后重新检查。
- `E_MANIFEST_SIGNATURE`、`E_ARTIFACT_SIGNATURE`：签名校验失败，重新获取官方文件；不跳过签名。
- `E_ARTIFACT_HASH`：文件完整性校验失败，重新下载。
- `E_HEALTH_*`、`E_ROLLBACK_*`：显示健康检查或回滚结果，并保留核心返回的日志路径。

更新诊断按钮只导出事务诊断 CLI 允许的脱敏状态和日志摘要，不导出用户文件内容或密钥。

## 兼容范围与限制

- 已安装在单一根分区的系统只能使用签名事务型 OTA； recovery ISO 的独立备份介质门禁不变。
- 26.3.2 必须先完成一次官方 bootstrap，bootstrap 的真实下载地址、SHA256、签名和公钥指纹见官方发布页。
- 不支持通过修改 GRUB、关闭签名校验、绕过空间检查或删除 `/home` 来“强制更新”。
- 发生健康检查失败时，系统按事务状态机自动回滚；同一块磁盘损坏不在保护范围内。
