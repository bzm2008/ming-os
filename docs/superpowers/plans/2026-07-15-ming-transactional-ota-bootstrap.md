# Ming Transactional OTA Bootstrap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

**Goal:** Give a bootstrapped Ming OS client a signed, rollback-capable major update path that preserves /home on a single root partition without Calamares or repartitioning.

**Architecture:** A one-time signed DEB bootstrap installs a transaction-aware updater, release trust key, initramfs hook, and post-boot health unit. The updater accepts only signed and version-compatible manifests, stages an allowlisted payload plus journal on the existing root filesystem, applies it during a one-time initramfs boot, and restores the journal before switch_root when health is not committed. ISO/Calamares remains recovery-only with the existing independent-media guard.

**Tech Stack:** Bash, Python 3 standard library, GnuPG gpgv, tar, zstd, sha256sum, initramfs-tools, systemd, GRUB, Node/Express, PHP, unittest, node:test.

def verify_detached_signature(manifest_path, signature_path, keyring, runner=subprocess.run):

def canonical_relative_path(value):

- [ ] **Step 3: Implement a four-state journal**

Use only these persistent states:

~~~text
prepared -> applying -> pending-health -> committed
                         |
                         -> rolling-back -> rolled-back
~~~

Stage each transaction at:

~~~text
/var/lib/ming-update/transactions/<release-id>/
~~~

Write state atomically:

~~~bash
write_state() {
    next="$1"
    temporary="$STATE_FILE.tmp.$$"
    printf 'state=%s\nattempt=%s\n' "$next" "$ATTEMPT" > "$temporary"
    sync -f "$temporary"
    mv -f -- "$temporary" "$STATE_FILE"
    sync -f "$(dirname "$STATE_FILE")"
}
~~~

Before each signed index operation, copy the old regular file to journal/files
or write a deletion marker, fsync the record, write a same-directory temporary
replacement, verify its hash, then rename it atomically. Reject anything not
in assets/ming-transaction-allowlist.txt. The payload builder accepts:

~~~bash
tools/build-ming-transaction-payload.sh --rootfs /absolute/rootfs --release-id 26.3.3-inplace-1 --out /absolute/out
~~~

It emits payload.tar.zst, content-index.tsv, content-index.sha256, and a
manifest fragment. It cannot walk /home or paths outside the allowlist.

- [ ] **Step 3: Implement the boot boundary**

The initramfs hook copies only ming-transaction-apply and required binaries
with copy_exec. local-top acts only if the kernel command line contains:

~~~text
ming.transaction=<release-id>
~~~

Implement this control flow:

~~~bash
case "$(ming-transaction-apply status --field state)" in
  prepared) ming-transaction-apply apply ;;
  pending-health) ming-transaction-apply recover ;;
  committed) exit 0 ;;
  *) ming-transaction-apply recover ;;
esac
~~~

The health service uses TimeoutStartSec=90 and only commits when transaction
state is pending-health, graphical.target and LightDM are active, and
ming-desktop-healthcheck reports a ready desktop. A health failure leaves
pending-health untouched so the next boot rolls back.

modules/01_base.sh installs gpgv, zstd, tar, and initramfs-tools; it runs
update-initramfs -u -k all and checks the newest initramfs contains the
transaction runtime. modules/03_desktop.sh deploys and enables the health
unit.
