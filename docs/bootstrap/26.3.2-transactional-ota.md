# 26.3.2 单分区事务型 OTA Bootstrap 下载页

本页适用于仍运行 Ming OS 26.3.2、尚未具备 `transactional-slot-v1` 能力的机器。bootstrap 只需安装一次。它不会调用 Calamares、分区工具或格式化现有分区，也不会删除 `/home`。

## 发布负责人必须先填入真实值

以下字段由签名发布流水线生成并由核心负责人复核。尖括号内容是占位符，**占位值不可用于安装**：

| 项目 | 正式值 |
| --- | --- |
| 下载地址 | `<OFFICIAL_BOOTSTRAP_URL>` |
| SHA256 文件地址 | `<OFFICIAL_BOOTSTRAP_SHA256_URL>` |
| 签名文件地址（detached signature） | `<OFFICIAL_BOOTSTRAP_SIGNATURE_URL>` |
| 公钥指纹 | `<MING_OTA_RELEASE_FINGERPRINT>` |
| bootstrap 版本 | `<BOOTSTRAP_VERSION>` |

发布负责人必须替换上述占位符，并确认网页、GitHub Release 和 OTA discovery 返回相同的值。仓库没有真实产物时，不得猜测 SHA256 或指纹。

## 下载与校验

1. 只从官网 OTA 页面或对应的官方 GitHub Release 下载 DEB、`.sha256` 和 `.sig`。不要使用第三方镜像或论坛附件。
2. 将三个文件放在同一目录，先确认文件名和架构为 `amd64`。
3. 使用发布页提供的公钥导入到临时、只读的校验目录；不要导入私钥。
4. 在替换占位符后执行：

```sh
export BOOTSTRAP_DEB='ming-ota-bootstrap_<BOOTSTRAP_VERSION>_amd64.deb'
export BOOTSTRAP_SHA256='ming-ota-bootstrap_<BOOTSTRAP_VERSION>_amd64.deb.sha256'
export BOOTSTRAP_SIG='ming-ota-bootstrap_<BOOTSTRAP_VERSION>_amd64.deb.sig'
export RELEASE_KEYRING='/path/to/official-ming-ota-release-keyring.gpg'

test "${BOOTSTRAP_DEB}" != '<OFFICIAL_BOOTSTRAP_URL>'
sha256sum -c "${BOOTSTRAP_SHA256}"
gpgv --keyring "${RELEASE_KEYRING}" "${BOOTSTRAP_SIG}" "${BOOTSTRAP_DEB}"
```

`sha256sum -c` 必须报告匹配，`gpgv` 必须使用发布页公布的公钥指纹并以成功退出。任一检查失败都应删除下载文件并重新获取，**不跳过签名、不手工修改清单、不使用 `dpkg --force`**。

## 安装前检查

- 当前系统为 26.3.2、架构为 amd64，且有足够的根分区空间。
- 重要数据已有独立备份；事务型 OTA 的同盘回滚不能防护物理磁盘损坏。
- Secure Boot、管理员授权和电源稳定性满足发布页要求。
- 不要在 bootstrap 安装过程中关机或拔电。

校验通过后，由 Ming 设置中心或官方 bootstrap 安装说明触发一次安装。安装完成后，运行能力检测，必须看到 `transactional-slot-v1` 和对应 bootstrap 版本；能力检测失败时不得继续事务更新。

## 出现拒绝或失败时

保留设置中心显示的错误码、官方日志路径和脱敏诊断导出。`E_BOOTSTRAP_REQUIRED` 表示能力尚未安装，`E_BOOTSTRAP_SIGNATURE`/`E_BOOTSTRAP_VERSION` 表示 bootstrap 本身不可信或不兼容。不要改动 GRUB、initramfs、`/var/lib/ming-update` 或 `/home` 来绕过拒绝。

Recovery ISO 仍要求独立备份介质；本页的 bootstrap 不是恢复 ISO，也不会改变该门禁。
